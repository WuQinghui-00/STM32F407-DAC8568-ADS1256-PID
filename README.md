# STM32F407 电压闭环控制系统

基于 STM32F407VET6 的嵌入式闭环控制系统，集成 DAC8568（16位DAC）和 ADS1256（24位ADC），实现高精度电压输出与采样，支持 PID 控制、波形生成和上位机通信。

---

## 硬件架构

### 主控
- **MCU**: STM32F407VET6 (Cortex-M4, 168MHz)

### 外设连接
| 外设 | 接口 | 引脚 |
|------|------|------|
| DAC8568 | 软件 SPI | PA5(SCK), PA7(MOSI), PB2(SYNC) |
| ADS1256 | 软件 SPI | PD0(SCLK), PD1(DIN), PD2(DOUT), PD3(DRDY), PD4(CS), PD5(RST) |
| USART1 | UART | PA9(TX), PA10(RX) |

### 电源
- DAC8568: 5V 供电
- ADS1256: 5V 供电
- STM32: USB 供电

---

## 功能特性

- **DAC8568 驱动**: 16 位分辨率，软件 SPI 模拟 Mode 3，输出范围 0-5V
- **ADS1256 驱动**: 24 位分辨率，软件 SPI 模拟 Mode 1，100SPS 采样率
- **PID 控制**: 位置式 PID 算法，带积分限幅和参数变化自动复位
- **前馈补偿**: 基于开环标定曲线（ADC = 0.972 × DAC + 0.197）消除系统偏置
- **波形生成**: 正弦波、方波、三角波输出
- **上位机协议**: 36 字节接收帧，66/82/86 字节回复帧，DMA + IDLE 中断接收
- **上位机工具**: Python 脚本，支持交互、监控、标定、波形、实时绘图

### 核心模块

#### 1. DAC8568 驱动
写通道数据流：计算码值 → 打包 32 位命令 → 拉低 SYNC → SPI 发送 4 字节 → 拉高 SYNC → DAC 输出

初始化顺序：复位 → 上电 → 使能内部基准（关键步骤，否则 REF 引脚为 0V）

#### 2. ADS1256 驱动
读取数据流：切换通道 → 启动转换 → 等待 DRDY 变低 → 读取 24 位数据 → 换算电压

#### 3. PID 控制器
输出 = P × 误差 + I × 积分 + D × 微分 + 前馈

积分限幅防止饱和，参数变化时自动复位积分。

#### 4. 通信协议
接收帧（PC → STM32）36 字节：
- 字节 0: 0x68（帧头）
- 字节 5: 命令码
- 字节 10-13: data1（大端序）
- 字节 14-17: data2（大端序）
- 字节 18-21: data3（大端序）
- 字节 34: 校验和（0xFF - 累加和）
- 字节 35: 0x16（帧尾）

回复帧：
- 66 字节：标准 ACK
- 82 字节：采集数据（ADC/DAC/温度）
- 86 字节：设备参数（PID/波形设置）

#### 5. 上位机工具 (test_host.py)
运行模式：
- `-m interactive`: 交互式命令行
- `-m quick`: 快速测试
- `-m monitor`: 持续监控
- `-m calibrate`: 开环标定
- `-m wave`: 波形测试
- `-m plot`: 实时绘图（matplotlib）

交互命令：
| 命令 | 功能 |
|------|------|
| 6 `<mV>` | 设置目标电压 |
| 7 `<P> <I> <D>` | 设置 PID 参数 |
| 5 | 查询实时数据 |
| 1 | 查询设备参数 |
| w `<mode>` | 设置波形模式（0=关,1=正弦,2=方波,3=三角波） |
| f `<mHz>` | 设置波形频率 |
| o `<mV>` | 设置波形偏置 |
| a `<mV>` | 设置波形幅度 |
| p | 启动/停止绘图 |
| q | 退出 |

## 快速开始

### 1. 编译烧录
用 Keil 打开 `MDK-ARM/stm32f407com.uvprojx`，按 F7 编译，F8 烧录。

### 2. 安装 Python 依赖
```bash
pip install pyserial matplotlib
```

### 3. 启动上位机
```bash
python test_host.py COM9 -m interactive
```

### 4. 使用示例
```bash
# 设置目标电压 1.5V
6 1500

# 设置 PID 参数 P=2.0, I=0.2, D=0.0
7 2.0 0.2 0.0

# 查询实时数据
5

# 输出正弦波，频率 1Hz，幅度 1.0V，偏置 0V
w 1
f 1000
a 1000
o 0

# 启动实时绘图（需先安装 matplotlib）
p
```

## 调试记录

### 已解决的问题

| 问题 | 原因 | 解决 |
|------|------|------|
| DAC REF 引脚为 0V | 半字节交换错误 | 修正命令字节（0x80→0x08, 0x70→0x07, 0x40→0x04） |
| DAC0 输出为 0V | 测错引脚（测了 DAC1） | 将 8 个通道全测一遍 |
| ADS1256 ID 为 0x0F | 供电不足（3.45V） | 改接 5V 供电 |
| ADS1256 VCC 仍偏低 | GND 经过 DAC 中转 | 所有 GND 直接并联到电源 |
| PID 收敛慢（几十秒） | DAC 和 ADC 有固定偏置 | 加入前馈补偿 |
| 波形幅度为 0 | 缺少幅度命令 | 新增 CMD_WAVE_AMP = 0x24 |
| 波形不平滑 | 采样率太低 | 30SPS → 100SPS |

### 经验总结
1. REF 引脚是 DAC8568 的第一诊断点：2.5V 表示基准使能成功
2. 5V 器件必须给足 5V，3.3V 供电是坑
3. GND 必须直接接到电源，不要经过其他模块中转
4. SPI 通信先确认模式（Mode 1 最常见）
5. 测不到信号时，把所有引脚测一遍（可能测错了）
6. 前馈是加速收敛的关键
